# 公众号写作工作流

> 本文档记录了标准的公众号文章创作流程，确保每次输出的文章风格统一、格式规范。

---

## 📋 创作触发词

用户输入以下任一内容即可开始创作：

- "写一篇关于 [主题] 的文章"
- "创作 [主题] 内容"
- "写 [主题] 公众号文章"

---

## 🎨 文章风格选择

系统支持两种写作风格，用户可选择或让系统根据内容自动判断：

### 风格 A：知鱼说（默认）

**特点**：实战派，不整虚的概念

| 项目 | 格式 |
|------|------|
| 开头 | 🐟 知鱼说：别被"..."这种假象骗了，今天咱只讲一个事儿——... |
| 结尾 | > 🐟 **关于知鱼**<br>> 专注 AI 赛道的实战派，不整虚的概念，只讲能落地的玩法。 |
| 章节 | 🔥📊💡🎯⚡ emoji 标记 |
| 口语化 | "坦率的讲"、"说实话"、"怎么说呢" |
| 排版 | 引用块开头结尾，重点加粗，无小标题 |

### 风格 B：卡兹克

**特点**：有见识的普通人在认真聊一件打动他的事

| 项目 | 格式/特点 |
|------|-----------|
| 开头 | 从具体事件/场景切入，如"最近这两天，被...给刷屏了" |
| 结尾 | 固定尾部（三连/星标）+ 个人署名 |
| 价值观 | 永远对世界保持好奇；讲人话，像个活人；真诚是唯一的捷径 |
| 章节 | 口语化转场，无小标题，靠节奏推进 |
| 口语化 | "这玩意"、"不是哥们"、"太牛逼了"、"我当时就愣住了" |
| 标点禁令 | 禁用冒号、破折号、双引号（用「」替代） |
| 排版 | 无小标题，一句话可独立成段，长短句交替 |

---

## 🎯 风格选择建议

| 场景 | 推荐风格 | 理由 |
|------|----------|------|
| 行业分析、趋势解读 | 知鱼说 | 更有条理，适合分析类内容 |
| 产品体验、调查实验 | 卡兹克 | 更有故事感，适合叙事类内容 |
| 方法论、工具分享 | 知鱼说 | 更清晰，适合教学类内容 |
| 个人观点、情绪表达 | 卡兹克 | 更真实，适合观点类内容 |

## 📐 通用排版格式规范

### 必须遵守的规则

| 项目 | 规范 |
|------|------|
| 小标题 | **加粗文字** 或口语化转场 |
| 列表 | 用文字描述，不用 - 符号 |
| 数字列表 | 第一、第二、第三 |
| 分隔线 | 用 --- |
| 重点内容 | **加粗** |

## 🔍 搜索引擎说明

| 引擎 | API Key | 搜索方式 | 擅长场景 |
|------|---------|----------|----------|
| **Tavily** | `TAVILY_API_KEY` | 关键词搜索 | 综合搜索、新闻、财经 |
| **Exa** | `EXA_API_KEY` | 语义搜索 | 学术论文、技术文档、高质量内容发现 |
| **双引擎（默认）** | 两个都配置 | Tavily + Exa | 全面覆盖，互补优势 |

### 通用规则

| 项目 | 规范 |
|------|------|
| 数量 | 3 张 |
| 位置 | 第1张 30%、第2张 60%、第3张 90%（按文章行数占比） |
| 比例 | 16:9 |
| 格式 | <!-- IMAGE_N: [英文 prompt] --> |

**配图位置详解**：

| 图片 | 位置 | 作用 | 放置时机 |
|------|------|------|----------|
| IMAGE_1 | 文章进度 30% | 吸引注意力，配合正文展开 | 正文第一个核心观点展开之后 |
| IMAGE_2 | 文章进度 60% | 缓冲节奏，配合转折或深入分析 | 文章中段转折处（如泼冷水、反面论证） |
| IMAGE_3 | 文章进度 90% | 收尾升华，配合总结判断 | 结论/判断部分，结尾引用块之前 |

### 不同风格的配图差异

| 风格 | 配图风格 | Prompt 示例 |
|------|----------|------------|
| 知鱼说 | 信息图表、流程图 | clean diagram, minimalist tech style |
| 卡兹克 | 场景化、故事化 | narrative scene, character-focused, digital art style |

## 📝 写作步骤

### Step 0: 选择风格（新增）

根据内容类型和目标读者选择写作风格，或使用默认风格（知鱼说）。

### Step 1: 确认主题

接收用户的创作主题。

### Step 2: 搜索素材（可选）

**双引擎搜索**（Tavily + Exa）：

```bash
# 双引擎搜索（推荐，采集更全面）
py ai-wechat-publisher/scripts/research.py research "[主题]" --engine all --save output/research_notes.md

# 仅 Tavily（综合搜索）
py ai-wechat-publisher/scripts/research.py research "[主题]" --engine tavily --save output/research_notes.md

# 仅 Exa（语义搜索，擅长高质量内容发现）
py ai-wechat-publisher/scripts/research.py research "[主题]" --engine exa --save output/research_notes.md

# Exa 精细搜索（按分类、日期过滤）
py ai-wechat-publisher/scripts/research.py exa-search "[主题]" --category "news" --start-date "2025-01-01" --with-summary
```

**引擎对比**：

| 特性 | Tavily | Exa |
|------|--------|-----|
| 搜索方式 | 关键词搜索 | 语义搜索 |
| 擅长领域 | 综合、新闻、财经 | 学术、技术、高质量内容 |
| AI 摘要 | ✓ | ✓ |
| 分类过滤 | 通用/新闻/财经 | company/paper/news/github/pdf 等 |
| 日期过滤 | ✓ | ✓ |
| 适合场景 | 广泛搜索 | 精准发现 |

### Step 3: 撰写文章

按照选定风格的规范创作。

**知鱼说风格检查点**：
- [ ] 开头有「知鱼说」引用块
- [ ] 正文用 emoji 章节标记（🔥📊💡🎯⚡）
- [ ] 结尾有「关于知鱼」引用块
- [ ] 3 个 IMAGE_N 标记（位置：30% / 60% / 90%）
- [ ] 重点内容用 **加粗**

**卡兹克风格检查点**：
- [ ] 开头从具体事件/场景切入
- [ ] 口语化转场，无小标题
- [ ] 禁用冒号、破折号、双引号
- [ ] 至少 8-10 个口语化词组
- [ ] 有自嘲或承认不足
- [ ] 有文化升维或哲思
- [ ] 固定尾部 + 个人署名

### Step 4: 质量检查（四层自检体系）

**所有风格统一使用四层质检体系**，逐层通过后才算达标。

#### L1 硬性规则检查（自动扫描层）

**L1-1 禁用词扫描**，全文搜索以下词汇，出现则必须替换：
- "说白了" → "坦率的讲"、"其实就是"
- "意味着什么" / "这意味着" → "那结果会怎样呢"、"所以呢"
- "本质上" → "说到底"、"其实"
- "换句话说" → "你想想看"、"也就是说"
- "不可否认" → 直接删掉，换成正面陈述
- "综上所述" / "总的来说" → 换成具体的回扣句
- "首先...其次...最后" → 用自然的转场词替代
- "值得注意的是" / "不难发现" → 删掉，直接说

**L1-2 禁用标点扫描**，全文搜索以下标点，出现则必须替换：
- 冒号"：" → 用逗号替代
- 破折号"——" → 用逗号或句号替代
- 双引号""或"" → 用「」替代，或直接不加引号

**L1-3 结构性套话扫描**：
- "让我们来看看..." / "接下来让我们..."
- "在当今...的时代" / "随着...的发展"
- 连续使用 bullet point 罗列观点（超过3个需改成散文叙述）
- 大段加粗（超过2行几乎肯定是过度结构化）

**L1-4 工具名检查**：确认所有提到的 AI 工具/产品都使用了具体名称。

#### L2 风格一致性检查（模式匹配层）

**L2-1 开头检查**：
- 是否从具体的、当下的事件/场景切入？（不是宏大叙事）
- 第一句话是否让读者产生"然后呢？"的冲动？

**L2-2 节奏与结构检查**：
- 是否有长短句交替？（连续3句以上句式长度相近 = 节奏呆板）
- 是否有一句话独立成段的"断裂"效果？（全文至少出现3次）
- 偏离主线的段落后面是否有"扣主线句"拉回来？

**L2-3 口语化检查**：
- 是否使用了口语化词组？（全文至少8-10个不同的口语化表达）
- 有没有至少一处自嘲或承认不足？

**L2-4 标点禁令二次确认**：全文是否完全没有冒号、破折号和双引号？

#### L3 内容质量检查（深度审查层）

**L3-1 观点支撑检查**：每个核心观点是否都有具体的人/场景/细节/数据支撑？

**L3-2 知识输出方式检查**：
- 知识点是否以"聊着聊着顺手掏出来"的方式呈现？
- 有没有出现"下面我来介绍"这种教科书式科普？

**L3-3 文化升维检查**：有没有至少一处从具体事件连接到更大的文化/哲学/历史参照物？

**L3-4 对立面与同理心检查**：
- 在讲核心观点时，是否有对对方立场的理解和承认？
- 有没有先站到读者的处境里，再给出自己的视角？

**L3-5 文章类型专项检查**：根据文章类型做针对性检查。

#### L4 活人感终审（最终人格层）

以读者视角通读全文，回答一个核心问题：

**"读完这篇文章，我感觉是一个真人在跟我聊天，还是一个 AI 在给我输出信息？"**

**L4-1 温度感**：情绪表达是体感记忆还是知识性描述？
**L4-2 独特性**：这篇文章是否有"只有这个作者才会写出来的角度"？
**L4-3 姿态检查**：有没有滑入"导师教学生"或"品牌做营销"的姿态？
**L4-4 心流检查**：从头到尾读，有没有哪个地方注意力断掉了？

#### 质检输出格式

完成四层自检后，输出质检报告：

```
## 质检报告

**L1 硬性规则** ✅/❌
- 禁用词：X处命中（已修复/待修复）
- 禁用标点：X处命中（已修复/待修复）
- 结构套话：X处命中（已修复/待修复）
- 空泛工具名：X处（已修复/待修复）

**L2 风格一致性** ✅/❌
- 开头：✅/❌
- 节奏：✅/❌
- 口语化：✅/❌（使用了X个口语词组）
- 标点二次确认：✅/❌

**L3 内容质量** ✅/❌
- 观点支撑：✅/❌
- 知识输出：✅/❌
- 文化升维：✅/❌
- 对立面与同理心：✅/❌

**L4 活人感** ✅/❌
- 温度感：✅/❌
- 独特性：✅/❌
- 姿态：✅/❌
- 心流：✅/❌

**总评**：4层全部通过 / X层需要返工
```

## ✅ 质量检查清单

发布前检查（**所有风格统一使用四层质检**）：

### L1 硬性规则

- [ ] 无禁用词（说白了、意味着什么、本质上、换句话说、不可否认）
- [ ] 无禁用标点（冒号、破折号、双引号 → 用「」替代）
- [ ] 无结构套话（让我们来看看、随着...的发展）
- [ ] 所有工具名具体（无"AI工具""某个模型"）

### L2 风格一致性

- [ ] 开头从具体事件/场景切入
- [ ] 有长短句交替（至少3处单句独立成段）
- [ ] 口语化词组至少8-10个
- [ ] 有自嘲或承认不足

### L3 内容质量

- [ ] 每个核心观点有具体场景/数据支撑
- [ ] 知识自然带出，非教科书式科普
- [ ] 有文化升维（连接更大的参照物）
- [ ] 有对立面的理解和同理心

### L4 活人感

- [ ] 有体感记忆式的情绪表达
- [ ] 有独特角度（不是换个AI也能写出来的）
- [ ] 姿态是"真人在聊天"而非"导师在教学生"
- [ ] 全文心流顺畅，无断点
